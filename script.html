The Problem

Students and professionals often lack the time to write detailed project descriptions for their portfolios. This friction results in outdated websites and missed opportunities. There’s no current solution for productive, on-the-go content creation, meaning valuable experiences are often poorly documented.

Takeaway: Professionals lose opportunities because documenting projects is time-consuming and difficult to do on the go.

The Solution

Foligo turns dead time, like walking between classes, into productive portfolio-building sessions. A user starts a conversation with our AI assistant, which asks intelligent, clarifying questions. From that conversation, Foligo instantly generates a detailed portfolio write-up, a LinkedIn post, and a concise update for X (formerly Twitter).

Takeaway: Foligo turns "dead time" into productive sessions, generating three distinct content types from one conversation.

Challenges and Solutions

Developing a multi-turn conversational AI that maintains context was a major challenge, requiring careful prompt engineering for the Gemini API. We needed the agent to ask intelligent, clarifying questions and remember previous answers to build a coherent project summary.

Another significant hurdle was real-time voice. We pivoted to the ElevenLabs API to handle low-latency speech-to-text and text-to-speech, which was critical for a natural conversational feel.

Takeaway: We overcame conversational context challenges with prompt engineering and pivoted to the ElevenLabs API for real-time voice.

Accomplishments

We are proud of creating a functional, end-to-end application that solves a real-world problem. Successfully integrating multiple complex APIs (Gemini for intelligence, ElevenLabs for voice) into a single, seamless user experience is a major accomplishment.

We learned the nuances of prompt engineering for LLMs to control tone, structure, and detail. Our team also gained valuable experience with the practical challenges of implementing real-time voice AI and building a robust RESTful API to support a complex, stateful workflow.

Takeaway: We gained practical experience in prompt engineering for LLMs and building robust REST APIs for voice AI.

A Note on Infrastructure

Going into this challenge, we planned to create our own speech-to-speech agent using Vultr, Digital Ocean, and open-source models. However, we were denied GPU access from both AI companies and spent hours with support only to be told we could not use any high-compute resources.

The judges mentioned they would take this into consideration if we used their services somewhere. Despite the setback, we still hosted our onboarding pages and other portions of the site on their CPU servers (using a LAMP stack) and pivoted to the ElevenLabs API for the voice interaction.

Accessibility

Our voice-first design is central to our commitment to accessibility. It provides a powerful alternative to traditional keyboard interfaces, making portfolio creation accessible to users with motor impairments or those who find extensive typing difficult.
